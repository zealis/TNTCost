# 货币汇率转换功能实现计划

## 1. 仓库分析

### 1.1 现有代码结构
- **账户模型**：`lib/models/account.dart` 包含账户信息，已添加 `currency` 字段
- **货币工具**：`lib/utils/money_utils.dart` 提供货币格式化功能
- **资产页面**：`lib/views/account_page.dart` 显示账户列表和总余额
- **设置页面**：`lib/views/settings_page.dart` 包含本币设置

### 1.2 当前总余额计算逻辑
- 在 `account_page.dart` 第280行，简单相加所有账户余额，未考虑货币汇率
- 显示时使用 `MoneyUtils.formatMoney(totalBalance)`，默认使用人民币符号

### 1.3 货币支持
- 当前支持的货币：CNY、USD、EUR、GBP、JPY
- 需要增加更多流行货币

## 2. 实现计划

### 2.1 创建汇率服务

#### 2.1.1 选择汇率API
- **默认使用Frankfurter**（免费，支持170+货币，由欧洲中央银行提供数据）
- **备用API**：ExchangeRate-API（作为兜底方案）
- 优势：双重API机制提高可靠性，Frankfurter无API密钥需求，数据可靠

#### 2.1.2 实现汇率服务类
- 创建 `lib/services/exchange_rate_service.dart`
- 实现以下功能：
  - 从API获取汇率数据
  - 本地缓存汇率数据（使用SharedPreferences）
  - 检查汇率数据是否需要更新（每日更新）
  - 提供汇率转换方法
  - 实现错误处理和兜底机制

#### 2.1.3 汇率数据结构
- 缓存格式：
  ```json
  {
    "lastUpdated": "2026-04-19T12:00:00Z",
    "base": "CNY",
    "rates": {
      "USD": 0.138,
      "EUR": 0.127,
      "GBP": 0.109,
      "JPY": 20.15,
      ...
    }
  }
  ```

## 3. 依赖和考虑因素

### 3.1 依赖项
- `http`：用于API请求
- `shared_preferences`：用于缓存汇率数据
- `intl`：用于日期格式化

### 3.2 风险和应对
- **API限流**：ExchangeRate-API免费版有请求限制，需要合理缓存
- **网络连接**：实现离线缓存和兜底机制
- **汇率波动**：明确显示汇率更新时间，避免误导用户
- **用户体验**：汇率更新时显示加载状态，避免界面卡顿

### 3.3 性能优化
- 汇率数据只在需要时更新
- 缓存机制减少网络请求
- 异步计算总余额，不阻塞UI

## 4. 预期效果

- 资产页面总余额显示转换为本币后的金额
- 支持多种流行货币
- 汇率自动每日更新，用户也可手动刷新
- 网络不可用时使用缓存汇率，缓存失效时显示原币总余额
- 本币设置界面更加美观，使用弹窗形式

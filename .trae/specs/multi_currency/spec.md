# 多币种功能 - 产品需求文档

## 概述
- **Summary**: 为TNT Cost应用添加多币种支持，允许用户在不同账户使用不同货币，并在交易时自动显示对应账户的货币单位。
- **Purpose**: 解决用户在管理不同货币资产时的需求，提供更灵活的财务管理功能。
- **Target Users**: 需要管理多币种资产的用户，如国际旅行者、跨国工作者等。

## 目标
- 支持为不同账户设置不同的币种
- 在设置界面中添加本币设置选项
- 在添加和编辑账户时增加币种选择功能
- 在添加交易时，根据所选账户的币种显示对应货币单位
- 保持现有功能的兼容性

## 非目标（不在范围内）
- 货币汇率转换功能
- 多币种报表统计
- 自动汇率更新

## 背景与上下文
- 当前应用只支持单一货币（人民币）
- 账户模型中没有币种字段
- 货币格式化工具类硬编码使用人民币符号
- 用户无法为不同账户设置不同币种

## 功能需求
- **FR-1**: 在账户模型中添加币种字段，支持多种货币
- **FR-2**: 在设置界面中添加本币设置选项
- **FR-3**: 在添加和编辑账户时增加币种选择功能
- **FR-4**: 在添加交易时，根据所选账户的币种自动显示对应货币单位
- **FR-5**: 货币格式化工具类支持根据币种显示不同的货币符号

## 非功能需求
- **NFR-1**: 保持应用性能不受影响
- **NFR-2**: 界面设计符合现有应用风格
- **NFR-3**: 向后兼容，现有数据无需迁移

## 约束
- **技术**: Flutter框架，Isar数据库
- **业务**: 保持现有功能的完整性
- **依赖**: 无额外依赖

## 假设
- 应用将支持常见货币（如USD、EUR、GBP、JPY等）
- 本币设置默认为人民币
- 账户币种默认为本币

## 验收标准

### AC-1: 账户模型添加币种字段
- **Given**: 打开应用
- **When**: 查看账户数据结构
- **Then**: 账户模型应包含币种字段
- **Verification**: `programmatic`

### AC-2: 设置界面添加本币设置
- **Given**: 进入设置界面
- **When**: 查看设置选项
- **Then**: 应看到本币设置选项
- **Verification**: `human-judgment`

### AC-3: 添加账户时可选择币种
- **Given**: 进入添加账户界面
- **When**: 填写账户信息
- **Then**: 应看到币种选择选项
- **Verification**: `human-judgment`

### AC-4: 编辑账户时可修改币种
- **Given**: 进入编辑账户界面
- **When**: 修改账户信息
- **Then**: 应看到币种选择选项并可修改
- **Verification**: `human-judgment`

### AC-5: 添加交易时显示对应账户的货币单位
- **Given**: 进入添加交易界面
- **When**: 选择不同币种的账户
- **Then**: 金额输入框应显示对应账户的货币单位
- **Verification**: `human-judgment`
